Optimal Page Replacement

Theory

In this algorithm, OS replaces the page that will not be used for the longest period of time in future.

Important points about Optimal page replacement algorithm:
1. The consequence of the revelation of Belady’s Anamoly.
2. Most reduced page fault rate of all algorithms and will never suffer from Belady’s Anamoly.
3. Just it replaces the pages that won’t be utilized for the longest time frame.
4. Optimal page replacement is accurate as possible required, but not possible in practice as the operating system cannot know future requests.
5. The utilization of optimal Page replacement is to set up a benchmark so that other replacement algorithms can be analyzed against it.

Advantages of Optimal page replacement

  • It is free from Belady’s Anamoly.
  • Most reduced page fault rate
  • Twice tantamount to FIFO Page Replacement Algorithm

Disadvantages of Optimal Page Replacement

  • Optimal Page Replacement is not possible in practice as the operating system cannot know future requests. The use of Optimal Page replacement is to set up a benchmark so that other replacement algorithms can be analyzed against it.
  • It is hard to execute

Example

  1. Initially all slots are empty, so when 7 0 1 2 are allocated to the empty slots —> 4 Page faults
  2. 0 is already their so —> 0 Page fault.
  3. when 3 came it will take the place of 7 because it is least recently used —>1 Page fault
  4. 0 is already in memory so —> 0 Page fault.
  5. 4 will takes place of 1 —> 1 Page Fault
  6. Now for the further page reference string —> 0 Page fault because they are already available in the memory.
